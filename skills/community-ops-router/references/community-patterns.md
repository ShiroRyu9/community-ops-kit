# Community Patterns

Use these as transferable hypotheses when current project evidence is limited. They can suggest questions or options, but they do not supply project rules, configuration, or facts.

## Community Types

| Type | Main job | Typical loop | Common failure |
|---|---|---|---|
| Product/customer | Support, education, retention, feedback | question -> answer -> accepted solution -> product insight | confusing support with engagement |
| Creator/UGC | Creation, feedback, recognition | prompt -> submission -> feedback -> showcase -> collaboration | collecting content without permission or reuse path |
| Paid member | Exclusive value and trust | access -> benefit use -> support/activity -> renewal signal | overpromising benefits |
| Developer/open source | Onboarding and contribution | docs -> first issue -> review -> recognition -> maintainer path | ignoring governance and maintainer load |
| Real-time chat | Activation and fast feedback | join -> first action -> discussion/activity -> recognition -> repeat | noisy surfaces without searchable memory |
| Forum/knowledge | Searchable answers and durable content | question -> answer -> moderation -> accepted answer -> knowledge base | treating durable knowledge as disposable chat |

## Lifecycle Lens

```text
discovery -> entry -> first action -> repeat participation -> contribution -> recognition -> retention/upgrade -> review
```

At each stage define the user segment, next action, surface, evidence, feedback or support, and review decision.

## Common Mechanisms

- onboarding: one clear first action;
- activation: a low-friction prompt, poll, checklist, starter task, or submission;
- participation: recurring rituals, events, office hours, challenges, AMAs, or discussions;
- contribution: answers, tutorials, tests, reports, templates, or user work;
- recognition: showcases, featured posts, roles, badges, or contributor paths;
- knowledge reuse: FAQs, accepted answers, tutorials, case studies, or product docs;
- governance: rules, moderation, escalation, appeals, permissions, and evidence retention;
- business link: activation, retention, support efficiency, product feedback, content assets, or paid value.

Treat these as hypotheses until project evidence confirms them.
