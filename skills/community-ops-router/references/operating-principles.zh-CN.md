# 运营原则

[English](operating-principles.md)

当 Router 自带的流程不足以支撑准确判断时，读取本参考。按顺序应用以下规则，不要跳过证据分类直接给出行动。

## 运营规则

### 1. 先确认依据，再下结论

- 判断问题前，先检查已有数量、时间、材料、用户反馈和历史结果。
- 影响决策的证据需要记录来源和观察周期。
- 重要陈述分成四种状态：
  - **已确认**：有明确来源或经过验证的观察直接支持。
  - **假设**：解释合理，但仍需验证。
  - **数据缺口**：当前缺少影响判断的信息。
  - **待审批**：动作已经准备，需要人工授权。
- 没有发现证据，不能直接说明某件事不存在。
- 证据较弱时，缩小结论范围，并给出验证方法。

### 2. 找出主要卡点

- 找到目标用户失败、停住、重复操作或离开的准确步骤。
- 按“用户群体 + 预期动作 + 已观察到的阻碍 + 证据”描述卡点。
- 根据用户影响、出现频率、可逆性和证据可信度排列多个问题。
- 每个周期只选一个主要卡点，其他问题标记为次要或尚未验证。
- 无法确定主要卡点时，先收集最小必要数据，再设计运营方案。

### 3. 选择最小有效动作

- 选择最可能改变目标行为、同时能产生可观察信号的最小改动。
- 明确一个目标人群、一个下一步动作、一个负责人和一个复盘点。
- 影响或证据不确定时，优先运行可撤回的小范围试验。
- 执行前写清预期信号、失败条件和停止条件。
- 只有试验确认机制有效，或暴露出明确的新限制后，才扩大范围。

### 4. 每个阵地只承担一个主要任务

- 给每个频道、论坛、活动、消息、表单或报告分配一个明确任务。
- 用用户动作描述这个任务，例如提问、投稿、学习、决策或复盘。
- 无关动作应转到其他阵地，不要继续往同一位置叠加说明。
- 明确该阵地应产生什么证据，以及由谁审核。
- 一个阵地存在多个互相竞争的行动提示时，先简化，再增加流量或内容。

### 5. 自动化不能越过边界

- 自动化可以收集、标准化、分类、汇总、准备提醒草稿和执行检查。
- 自动化可以准备预览、接收对象清单、计算结果、变更清单和审批包。
- 公开发布、私信、奖励、权限修改、Webhook 修改和官方声明必须经过人工明确确认。
- 审批只对当时展示的准确内容、受众、金额、权限或配置有效。
- 系统允许时，记录审批人、获批版本、执行结果和时间。
- 审批后目标、范围或风险发生变化时，停止执行并重新请求审核。

### 6. 每个动作都要能够复盘

- 按“触发 -> 动作 -> 证据 -> 响应 -> 复盘 -> 下一步决策”定义流程。
- 条件允许时，在改动前记录基线或对照点。
- 设置明确的复盘日期或触发事件，“以后再看”不算复盘规则。
- 记录实际发生的结果、仍不确定的信息，以及继续、修改或停止的决定。
- 把决定写回下一周期、文档或运营规则。

## 决策问题

推荐或路由工作前，检查以下问题：

1. 哪些事实已经确认，对应证据在哪里？
2. 哪些陈述是假设、数据缺口或待审批事项？
3. 受到影响的是哪类用户和哪个生命周期阶段？
4. 当前最影响用户完成下一步动作的阻碍是什么？
5. 哪个可撤回的最小动作可能减少这个阻碍？
6. 哪个可观察信号能支持或否定这个动作？
7. 该动作发生在哪个阵地，这个阵地的唯一主要任务是什么？
8. 自动化能否在不越过人工审批边界的情况下准备工作？
9. 谁在什么时间审核结果，之后需要做什么决定？

## 常见错误

| 错误做法 | 修正方式 |
|---|---|
| 根据一个个例或表面指标直接判断原因 | 标记为假设，再寻找行为或记录证据。 |
| 把所有可见问题都当成同等紧急 | 排列优先级，每个周期只处理一个主要卡点。 |
| 还没验证机制就启动大范围活动 | 先运行有明确信号、可以撤回的小范围试验。 |
| 让一个阵地同时承担新手引导、客服、推广和汇报 | 指定一个主要任务，把其他动作转到对应位置。 |
| 因为已有草稿，就允许自动化发送或修改敏感内容 | 展示准确动作，等待人工明确确认。 |
| 只汇报做了什么，没有后续决策 | 增加负责人、时间、证据和继续、修改或停止规则。 |
| 在同一建议中混写事实、猜测、缺失数据和审批事项 | 标记每种状态，并分别写清下一步。 |

## 输出检查

返回诊断、计划或复盘前，确认：

- [ ] 影响结论的证据已经标明来源和观察周期。
- [ ] 已确认事实、假设、数据缺口和待审批事项清楚分开。
- [ ] 已针对明确用户群体和生命周期阶段指出一个主要卡点。
- [ ] 建议动作是最小有效改动，并尽量可以撤回。
- [ ] 每个阵地只有一个主要任务和一个明确用户动作。
- [ ] 自动化范围只包含收集、整理、提醒准备、草稿和检查。
- [ ] 敏感执行仍处于待人工审批状态，没有写成已经完成。
- [ ] 成功信号、失败条件、负责人和复盘时间均已明确。
- [ ] 下一步决策明确为继续、修改、停止或补充证据。
- [ ] 任何结论都没有超过现有证据能够支持的范围。
