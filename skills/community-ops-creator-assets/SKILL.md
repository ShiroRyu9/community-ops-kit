# Community Ops Creator Assets

Use this skill when community members create value: works, answers, tutorials, demos, reviews, bug reports, templates, workflows, examples, or social proof.

## Creator Loop

```text
discover -> qualify -> ask/permission -> polish/package -> showcase -> follow-up -> asset library
```

## What To Extract

For each creator or submission, capture:

- who created it and where it appeared;
- what value it shows;
- whether attribution/permission is clear;
- the approved use, visibility, duration, and any revocation request;
- whether it is safe to showcase;
- where it can be reused: showcase, tutorial, FAQ, case study, social post, product example, onboarding proof;
- next follow-up.

## Guardrails

Require explicit confirmation before publishing or reusing member work, contacting or mentioning a creator, ranking submissions, or implying endorsement. Record permission scope and attribution; stop reuse when permission is revoked. Avoid shame rankings or personal profiling. Keep adult, private, sensitive, or paid-member content in the correct visibility context.

Creator recognition can support trust and continued contribution. Treat it as relationship work and review the evidence instead of assuming a retention result.
