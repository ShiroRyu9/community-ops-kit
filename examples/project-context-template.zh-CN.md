# 社区项目资料模板

只填写当前任务相关的部分。空白表示未知，Skill 应将其标记为数据缺口或待确认项，不能自行补成事实。

不要填写密码、Token、Webhook 地址、完整私信、支付资料或不必要的个人数据。需要引用敏感证据时，提供受控文件路径、记录 ID 或脱敏摘要。

## 基础资料

- 社区或产品：
- 平台：
- 目标用户：
- 当前问题或本次目标：
- 已有资料及来源：
- 负责人和审批人：
- 时区：

## 活动

- 活动目标：
- 参与人群和资格：
- 用户需要完成的动作：
- 开始时间、截止时间和时区：
- 活动入口和投稿位置：
- 投稿格式或有效证明：
- 评选、投票或反馈方式：
- 奖励或精选方式（已确认 / 待确认 / 无）：
- 提醒时间和发送位置：
- 安全、内容、年龄或地区限制：

## 新用户路径

- 用户从哪里进入：
- 当前欢迎页、规则页和引导入口：
- 角色、权限和访问规则：
- 希望用户完成的首次动作：
- 已知卡点和证据：
- 求助或客服入口：

## 创作者和内容资产

- 作品或贡献来源：
- 作者署名：
- 授权状态和允许用途：
- 可展示位置和受众：
- 筛选标准：
- 后续联系是否获批：

## 付费或高价值用户

- 用户分层定义：
- 已确认权益：
- 支付、资格、角色或访问依据：
- 当前问题或机会：
- 可用数据和证据：
- 联系、折扣、权益变更的审批边界：

## 日报或周复盘

- 日期范围和时区：
- 原始记录和数据来源：
- 已准备 / 已批准 / 已执行 / 已验证事项：
- 指标及统计口径：
- 已知数据缺口：
- 需要负责人决策的事项：

## 自动化

- 要重复执行的具体动作：
- 输入和唯一事实来源：
- 使用的 AI、定时任务、Bot、API 或平台工具：
- 已有权限范围：
- 运行频率：
- 人工确认点：
- 完成验证方式：
- 失败或结果不明时的停止条件：

## 客服和风险

- 问题类型和严重程度：
- 已确认事实：
- 截图、链接、日志、记录 ID 或政策依据：
- 缺失证据：
- 希望得到的处理结果：
- 可以公开的范围：

## 文案

- 读者：
- 发布位置：
- 已确认事实：
- 希望读者完成的动作：
- 语气和语言：
- 奖励、权益、数据、时间或官方声明的确认状态：
