# 示例：创作者挑战活动

[English](activity-challenge.md)

本示例只展示结构，其中的活动阵地、限制条件和活动信息不能作为其他社区的默认配置。

## 输入提示

```text
使用 community-ops-activity，为活跃创作者设计一个为期七天的挑战活动。

目标：收集有实际参考价值的产品案例，经过审核后可用于后续精选展示。
限制：不能自动公布获奖者，不能虚构奖励，未经创作者授权不能复用作品。
可用阵地：公告频道、投稿帖和每周精选帖。

发布前仍需确认：准确的开始和截止时间、时区、参与资格、有效投稿证明、评选方式、提醒时间和奖励状态。
```

## 预期输出

结果应包含：

- 目标人群和一个明确的参与动作；
- 活动入口、投稿证明、反馈、精选展示和后续动作；
- 发布、提醒、最后召集和结果公布草稿；
- 授权和奖励方面的数据缺口；
- 有效投稿数、首次参与人数、重复参与人数和可精选作品数等复盘证据；
- 发布活动、公布获奖者或承诺奖励前的人工确认点。

## 失败信号

- 只写“提升互动”，没有具体动作；
- 自行补充奖励或截止时间；
- 把草稿写成已经发布；
- 未经授权复用创作者作品；
- 从仓库示例中套用缺失的日期、奖励、规则或频道。
